Módulo de Retenciones sobre Aplicaciones de Anticipos a Proveedores

Manual paso a paso para replicar la solución en cualquier cuenta de NetSuite (Sandbox o Producción), con la descripción funcional de cada componente y script.

La solución automatiza la generación de retenciones fiscales (IR y Alcaldía) al momento de aplicar pagos anticipados a facturas de proveedores. Consta de un catálogo configurable, seis campos personalizados, una subficha y dos scripts. Siguiendo esta guía en orden, el módulo queda operativo en aproximadamente 60–90 minutos.

1Crear el catálogo de tipos de retención

Configuración → Personalización → Tipos de registro → Nuevo
  1. Nombre: Tipo de Retención NI · ID: _wh_tipo → Guardar.
  2. En la subficha Campos, crear: Etiqueta Tasa (%), ID _wh_tasa, tipo Número decimal.
  3. Crear el segundo campo: Etiqueta Cuenta Contable, ID _wh_cuenta, tipo Lista/Registro → Cuenta.
Importante: los IDs deben escribirse exactamente como se indica (NetSuite antepone customrecord/custrecord). El script busca estos IDs de forma literal.

2Poblar el catálogo

Desde el tipo de registro, usar Nuevo registro y crear las cinco entradas vigentes:

4Crear la subficha "Retenciones"

Configuración → Personalización → Subfichas → pestaña Transacción
  1. Título: Retenciones · ID: _retenciones · Primario: vacío → Guardar.
  2. Editar cada uno de los 5 campos del paso 3 → pestaña Mostrar → SUBFICHA = Retenciones.
  3. Opcional: usar Insertar antes para ordenar: checkbox → Retención 1 → 2 → 3 → Estimada.

5Subir los archivos de script

Documentos → Archivos → SuiteScripts → Añadir archivo

Subir los dos archivos de la carpeta retenciones_scripts:

  • ue_retencion_anticipo_v4.js — motor de servidor
  • cs_retencion_anticipo.js — validación de pantalla

6Registrar y desplegar el User Event Script

Configuración → Personalización → Secuencias de comando → Nuevo
  1. Seleccionar ue_retencion_anticipo_v4.js → Crear registro de script (NetSuite detecta "Evento del usuario").
  2. Nombre: UE - Retención sobre Anticipo NI · ID: _ue_wht_anticipo.
  3. Pestaña Parámetros, crear dos:
    • _wh_base_sin_iva · Casilla de verificación · Base sin IVA
    • _wh_tasa_iva · Número decimal · Tasa IVA (vacío = 15)
  4. Guardar → Desplegar script: Se aplica a Aplicación de pago anticipado al proveedor · Estado Pruebas · Nivel de registro Depurar.

¿Qué hace este script?

Se ejecuta en el servidor después de guardar la aplicación del anticipo (evento afterSubmit, solo en creación). Su flujo:

  1. Verifica que "Aplicar Retención" esté marcado y que exista al menos un tipo seleccionado; si no, termina sin hacer nada (queda constancia en el log).
  2. Consulta el catálogo para obtener tasa y cuenta de cada tipo elegido.
  3. Verifica que no exista ya un crédito para esta aplicación (anti-duplicado por externalid único WHT_PREPAYAPP_<id>).
  4. Recorre las facturas aplicadas (sublista bill) y calcula la retención de cada tipo sobre el monto aplicado (opcionalmente sobre base sin IVA).
  5. Crea un Crédito de Factura con una línea de gasto por tipo de retención —cada una contra su cuenta de pasivo— heredando fecha, moneda, subsidiaria y clasificación (departamento/clase/ubicación) de la aplicación o, en su defecto, de la factura.
  6. Aplica el crédito a las mismas facturas y guarda. Si otro proceso creó el crédito en paralelo, NetSuite rechaza el duplicado y el script lo registra como evento controlado.

7Registrar y desplegar el Client Script

Configuración → Personalización → Secuencias de comando → Nuevo
  1. Seleccionar cs_retencion_anticipo.js → Crear registro de script ("Secuencia de comandos de cliente").
  2. Nombre: CS - Retención sobre Anticipo NI · ID: _cs_wht_anticipo. No requiere parámetros.
  3. Desplegar: Se aplica a Aplicación de pago anticipado al proveedor · Estado Pruebas.

¿Qué hace este script?

Corre en el navegador mientras el usuario llena el formulario:

  • Bloqueo preventivo: los campos Retención 1/2/3 permanecen deshabilitados hasta marcar "Aplicar Retención"; al desmarcar, se limpian automáticamente.
  • Validación al guardar: impide guardar si la casilla está marcada sin tipo seleccionado, o sin facturas aplicadas con monto — elimina transacciones incompletas o "en blanco".
  • Cálculo estimado en vivo: actualiza "Retención Estimada" al instante cuando cambia el checkbox, los tipos o los montos aplicados, para que el usuario confirme el monto antes de guardar.
Sincronización: si se activa el parámetro "Base sin IVA" en el User Event, debe cambiarse la constante BASE_SIN_IVA = true dentro de este archivo para que el estimado coincida con el cálculo final.

8Pruebas de aceptación

  • Al abrir una aplicación nueva, los dropdowns aparecen deshabilitados; se habilitan al marcar la casilla.
  • Con C$2,000 aplicados e "IR 3%", la Retención Estimada muestra 60.00 en vivo.
  • Guardar con casilla marcada y sin tipo → mensaje de bloqueo (no guarda).
  • Al guardar correctamente, se genera el Crédito de Factura con una línea por tipo, cada una a su cuenta, aplicado a la(s) factura(s).
  • Editar y volver a guardar la misma aplicación → NO genera un segundo crédito (log: "Retención ya generada").
  • Aplicación sin casilla marcada → no genera nada (log: "Sin retención").
  • El reporte "CS - Auxiliar de Retenciones" refleja el crédito con su memo descriptivo.

9Publicación a usuarios

  1. En ambos despliegues (UE y CS): Estado Pruebas → Publicado.
  2. Definir la Audiencia: roles/empleados que registran aplicaciones de anticipos.
  3. Cambiar Nivel de registro del UE a Auditoría (evita saturar el log con mensajes de depuración).
  4. Para migrar de Sandbox a Producción: replicar esta guía manualmente o empaquetar los componentes con SuiteCloud Development Framework (SDF) / Copiar a cuenta.
Recordatorio operativo: en estado "Pruebas" los scripts solo se ejecutan para su propietario. Ningún otro usuario verá el comportamiento hasta publicar.
